# TDMS — Task Distribution and Management System

A personal task management system that breaks goals into time-bounded chunks,
tracks urgency and breach status, and syncs scheduled work to Google Calendar.

---

## Prerequisites

- Windows with WSL2 (Ubuntu) running
- Docker Desktop with WSL2 backend enabled
- Python 3.11+ inside WSL2
- Node.js LTS inside WSL2 (via nvm)

---

## First-Time Setup

### 1. Start the database

From the project root (inside WSL2):

```bash
docker compose up -d
```

Verify it is running:

```bash
docker ps
# You should see: tdms_db   postgres:15   Up
```

### 2. Set up the backend

```bash
cd backend

# Create and activate virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Copy the env file (only needed once)
cp ../.env.example .env

# Run database migrations
alembic upgrade head
```

### 3. Run the backend server

```bash
# Still inside backend/ with venv active
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Visit: http://localhost:8000/docs
You will see the full interactive API documentation (auto-generated by FastAPI).

### 4. Set up and run the frontend

Open a second WSL2 terminal tab:

```bash
cd frontend
npm install
npm run dev
```

Visit: http://localhost:5173
The TDMS web interface is now running.

## Running Tests

```bash
cd backend
source .venv/bin/activate
pytest
```

Tests use an in-memory SQLite database — no Docker required for testing.

## Google Calendar Integration (Optional)

Only do this after the rest of the app is working.

### Step 1 — Create Google Cloud credentials

1. Go to https://console.cloud.google.com
2. Create a project named "TDMS"
3. Enable the **Google Calendar API**
4. Go to APIs & Services → Credentials
5. Click **Create Credentials → OAuth 2.0 Client ID**
6. Application type: **Desktop App**
7. Download the JSON file
8. Save it as: `backend/google_credentials.json`

`google_credentials.json` is already in `.gitignore` — it will never be committed.

### Step 2 — Authorize

```bash
cd backend
source .venv/bin/activate
python -m app.services.calendar_auth
```

A browser window will open. Log in with your Google account and approve access.
A `token.json` file is saved. This is also gitignored.

### Step 3 — Verify

```bash
curl http://localhost:8000/calendar/status
# {"authorized": true, "message": "Google Calendar is connected."}
```

From this point, assigning a task chunk automatically creates a Google Calendar
event. Completing a chunk adds a ✓ prefix to the event. Failing a chunk deletes it.

## Daily Development Workflow

```bash
# Terminal 1 — database (only if not already running)
docker compose up -d

# Terminal 2 — backend
cd backend && source .venv/bin/activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# Terminal 3 — frontend
cd frontend && npm run dev

# Terminal 4 — tests (run whenever you change backend logic)
cd backend && source .venv/bin/activate && pytest
```

## Common Commands Reference

| Task | Command |
|---|---|
| Start DB | `docker compose up -d` |
| Stop DB | `docker compose down` |
| View DB logs | `docker logs tdms_db` |
| Run migrations | `alembic upgrade head` |
| Generate new migration | `alembic revision --autogenerate -m "description"` |
| Run tests | `pytest` |
| Run single test file | `pytest tests/test_tasks.py -v` |
| API docs | http://localhost:8000/docs |
